Quick Start Tutorial

This section provides a brief introduction to Procman and controlling processes with a deputy and a sheriff.

There are two types of programs in Procman - the deputy (bot-procman-deputy) and the sheriff (bot-procman-sheriff).

  • Deputies control processes, and there must be at least one deputy.
  • Sheriffs control deputies, and there can be at most one sheriff.

To get started, we'll show you how to run the deputy and the sheriff, and create and manage commands.

  • Starting a deputy and a sheriff
  • Creating a command
  • Managing commands
  • Saving and loading configurations

Starting a deputy and a sheriff

First, open up your favorite terminal program and run the deputy command:

bot-procman-deputy

Then, in another terminal, launch the sheriff GUI.

bot-procman-sheriff

When you run the sheriff, you should see a GUI that looks something like this:

bot-procman-sheriff with one deputy and no commands.

There are three panes in the sheriff GUI:

  • Top left: the command pane
  • Top right: the deputy pane
  • Bottom: the console pane

The command pane shows all the commands managed by Procman. Since we haven't created any commands, there isn't anything to see there yet.

The deputy pane shows all the deputies detected by the sheriff. We've started one deputy, so it shows up in the top right. Deputies are always named, and the default name is the computer hostname where the deputy process is running. In this case, the hostname is "contact", so the deputy is named "contact" as well.

The console pane shows console output (stdout and stderr) from running commands, and also status information from the sheriff.

Creating a command

To create a command, select the menu item "Commands -> New command".

Adding a command using the menu bar.

Following that, a dialog box should appear with several options to fill out.

Add command dialog.

The options are:

  • Deputy - which deputy should manage the command.
  • Command - the command to execute and any command-line arguments.
  • Name - a name for the command, used for display purposes and to identify the command.
  • Group - commands can be grouped together, or not.
  • Auto-restart - if checked, deputies will automatically restart commands when they terminate.

In the image above, we've set the command to xterm, a fairly common terminal emulator. If you don't have xterm installed, replace the command with something you do have on your system. We've also named the command "terminal emulator".

Managing commands

Now that we have a command, we can run it. Click on the command in the command pane so that it is highlighted. Once the command is selected, click on the menu bar item "Commands -> Start".

You should now see the xterm terminal emulator popup, and the status of the command should change to "Running".

And that's basically it.

You can stop, restart, move, and edit the selected command using the "Commands" menu.

There are a bunch more features to Procman, but creating, starting, and stopping processes is its core functionality.

Saving and loading configurations

It's often useful to save the commands you've created so that you can easily load them and run them again later on. To do this, select "File -> Save config as", and then select a filename.

The configuration files are human-readable, so if you save the configuration above and opened up the file in a text editor, you'd see something like the following:

cmd "terminal emulator" {
    exec = "xterm";
    host = "contact";
}

It should be pretty straightforward to figure out the format, and you can edit the configuration files directly with a text editor. For more information on the configuration file format, see the documentation on configuration files.

To load a configuration, click on the menu bar item "File -> load config".
